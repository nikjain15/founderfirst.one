# Deploy Guide — Penny Demo

Every deploy is: **build locally → copy dist to git tree → commit → push.**
GitHub Pages serves the built files from `penny/demo/` on the `main` branch.

---

## One-time setup (already done — skip if repo is initialized)

```bash
# From the workspace root (FounderFirst_Building Products/)
git init
git remote add origin git@github.com:nikjain15/founderfirst.one.git
git fetch origin main
git checkout origin/main -- .
git branch -M main
```

---

## Deploy after every change

Run all commands from the **workspace root** (`FounderFirst_Building Products/`).

### Step 1 — Build

```bash
cd BookKeeping/demo
npm run build
cd ../..
```

The output lands in `BookKeeping/demo/dist/`.

### Step 2 — Sync built files to the git tree

```bash
rsync -av --delete \
  BookKeeping/demo/dist/ \
  penny/demo/
```

This replaces only what changed. The `--delete` flag removes stale assets from old builds.

### Step 3 — Stage, commit, push

```bash
git add penny/demo/
git commit -m "deploy: <short description of what changed>"
git push origin main
```

GitHub Pages picks up the push automatically. Live at **founderfirst.one/penny/demo/** within ~30 seconds.

## What lives where

| Path | What it is |
|---|---|
| `BookKeeping/demo/` | Source (React + Vite). Edit this. Never commit it to git directly. |
| `BookKeeping/demo/dist/` | Build output. Generated by `npm run build`. Gitignored. |
| `penny/demo/` | What gets deployed. Always a copy of `dist/`. |
| `penny/demo/assets/` | Hashed JS + CSS bundles from Vite. |
| `penny/demo/config/` | `scenarios.json`, `industries.json`, `personas.json` — served at runtime. |
| `penny/demo/prompts/` | AI prompt files — served at runtime by the Cloudflare Worker client. |

---

## Deploying prompt or config changes only

If you only changed a file in `public/prompts/` or `public/config/` (no JS changes), you still need to run `npm run build` — Vite copies `public/` into `dist/` as part of every build.

---

## Verify before pushing

Open the preview build locally to confirm nothing is broken:

```bash
cd BookKeeping/demo
npm run preview   # serves dist/ at localhost:4173
```

---

## Git is at the workspace root

The `.git` folder lives at `FounderFirst_Building Products/` — **not** inside `BookKeeping/demo/`. Always run `git` commands from the workspace root.

```
FounderFirst_Building Products/   ← git root, run git commands here
├── BookKeeping/demo/             ← source, run npm commands here
└── penny/demo/                   ← deployed output, synced from dist/
```

## Troubleshooting

**Push rejected (non-fast-forward)**
```bash
git fetch origin main
git rebase origin/main
git push origin main
```

**SSH auth fails**
```bash
ssh -T git@github.com   # should print "Hi nikjain15!"
# If not, add your SSH key: ssh-add ~/.ssh/id_ed25519
```

**Site not updating after push**
Check the Pages deployment status at:
`https://github.com/nikjain15/founderfirst.one/actions`
