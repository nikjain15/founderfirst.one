Content Pipeline — Phase 1: Marketing Engine

Insights-driven, own-stack, brand-voiced content + audio. NotebookLM is not in the loop.
Updated 27 Jun 2026 Path: own-stack (auto, 1 human touch) Brains: Claude Voice: Chatterbox → ElevenLabs fallback Assembly: Podcastfy GPU: Fly, scale-to-zero

0 Stack decisions (locked)

  • Brains / synthesis & rewrite: Claude, tiered — Opus/Sonnet for Insights synthesis, Sonnet for brief→brand-voice rewrite. Ollama-on-Mac dropped.
  • Voice: Chatterbox (MIT, self-hosted) primary; ElevenLabs (Creator) fallback. Open-best as main, paid as fallback. Same locked reference clip clones in both.
  • Assembly: Podcastfy (Apache) — Claude script + Chatterbox/ElevenLabs voice.
  • Intake brain: the Analytics → Insights page becomes the demand sensor that feeds the pipeline.
  • GPU: new Fly GPU app (founderfirst-tts), scale-to-zero — spins up per job, stops idle. Same flyctl tooling as the Discord bridge.
  • NotebookLM: not wired in. Ad-hoc external tool only, by hand, for occasional hero pieces (e.g. investor podcast).

1 Prerequisite — land Insights on Claude

The Insights feature lives in the fix-web-flow worktree, not on main. Step zero:

  • Review + merge cleanly — run supabase migration list first, set aside any migrations that aren't ours, typecheck.
  • Swap synthesis off Ollama → Claude (Sonnet) before building anything downstream.

2 The loop (own-stack, fully automated)

idea→ drafting AUTO→ review YOU→ published AUTO

One human touch: your approval at review.

Removed from the earlier Path-A draft

  • NotebookLM packet generate-packet step
  • Upload outputs "awaiting outputs" upload form
  • packet column + packet_ready/awaiting_outputs statuses

What each stage does

  • idea — created by Insights "Send to pipeline" or manual entry; carries topic + angle + grounding.
  • drafting (auto) — content-draft fn: Claude writes grounded brief, brand-voice blog markdown, and an audio script; Podcastfy runs the script through the Chatterbox TTS interface → audio; SEO meta generated.
  • review (you) — preview rendered post + audio player → approve / edit / dismiss.
  • published (auto) — write blog markdown / content_pages row + create a promo row in email_schedules.

3 Data model (one migration)

content_pipeline
  id, source ('insight'|'manual'|'signal'), source_ref (insight_action_id, nullable),
  topic, angle, grounding (jsonb: metrics/snapshot that justified it),
  status ('idea'|'drafting'|'review'|'published'|'dismissed'),
  draft_md, script (jsonb), audio_url, seo (jsonb),
  published_ref (content_pages id / blog path), promo_schedule_id,
  created_by, created_at, updated_at

content_voice_profile          -- the single locked brand-voice asset
  id, name, reference_clip_url, provider_default ('chatterbox'),
  is_active, version, created_at

-- additions to existing insight_actions:
  action_type ('content'|'product'|'ops'|'kb'),
  resulting_content_id   -- closes the learning loop

RLS: admin-gated security-definer RPCs only (same pattern as insight_runs). All mutations audit-logged.

4 Insights → pipeline routing

  • Synthesis prompt tags each finding with action_type.
  • Content-typed action cards get a "Send to pipeline" button → creates a content_pipeline row (idea) pre-filled with topic + grounding.
  • Reuses existing card/button styling — tokens.css only, no inline values.

5 Admin surface — "Content Pipeline"

A board with stages idea → review → published (no upload UI). Reuses floating cards + version-history + live-toggle patterns from the email hub. Lives under Penny (pending decision Q1).

7 Brand & design adherence

  • Admin UI: tokens.css only; inherits admin patterns.
  • Blog output: markdown into existing VitePress theme → inherits typography/color free.
  • New sub-task — audio brand kit: on-brand audio-player embed + cover-art template, built from tokens. (The one thing blog text doesn't give for free.)
  • Voice grounded in the version-historied voice guide; brand voice = single locked asset (version-controlled like tokens.css).

8 Build sequence

  1. Land + de-Ollama Insights (Claude).
  2. Migration: content_pipeline, content_voice_profile, insight_actions columns.
  3. "Send to pipeline" routing from Insights.
  4. Content Pipeline admin tab (idea → review → publish board).
  5. content-draft fn (Claude, voice-grounded).
  6. Fly GPU Chatterbox + TTS interface + ElevenLabs fallback + Podcastfy.
  7. Audio brand kit.
  8. Publish → blog + email_schedules.
  9. End-to-end dry run on one real topic.

Process: one task per git worktree; small atomic commits; supabase/migrations/ is the only schema source of truth; verify every deploy from the system itself. (Per LEARNINGS.md / CLAUDE.md.)
