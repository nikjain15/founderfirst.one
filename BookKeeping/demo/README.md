# Penny Demo

A living, browser-based demo of Penny — an AI-first bookkeeper for US small business owners.

Built to give prospective users a realistic five-minute walkthrough of the product: onboarding, a first approval, the Penny conversation thread, and a financial summary. Feedback from this demo is what will shape the MVP scope.

**Live:** [founderfirst.one/penny](https://founderfirst.one/penny)
**Parent project:** [FounderFirst](https://founderfirst.one)
**Status:** v5 (in development). Supersedes the bundled v4 demo in `../tools/`.

---

## Design goals

1. **Feels real.** Production-grade typography, motion, and responsiveness. No "this is a mockup" disclaimers.
2. **Live AI voice.** Every Penny message is generated by Claude through a system prompt that enforces Penny's voice rules. No hard-coded copy in components.
3. **Token-efficient to build and iterate.** Copy lives in system prompts. Data lives in JSON. React components only render structure. Changing tone is a one-file edit.
4. **Guarded.** A rule-based validator sits between Claude and the screen. Banned phrases, disallowed emoji, and voice violations are caught and re-prompted.
5. **Public-safe.** No user data is persisted server-side. Demo state is ephemeral (per-session localStorage).

## How it works

```
  ┌─────────────┐     ┌──────────────────┐     ┌─────────┐
  │  React app  │────▶│ Cloudflare Worker│────▶│ Claude  │
  │  (browser)  │     │  (API proxy)     │     │  API    │
  └─────────────┘     └──────────────────┘     └─────────┘
         ▲                                           │
         │              ┌──────────────┐             │
         └──────────────│  Guardrails  │◀────────────┘
                        │  (validator) │
                        └──────────────┘
```

Every Penny utterance flows: intent + context → assembled system prompt → Claude → JSON response → validator → screen. Validation failures trigger a retry with the violation flagged.

**Two models on purpose.** Voice utterances (greetings, approval cards, capture parsing) run on **Claude Haiku** — fast enough to feel instant. My Books Q&A runs on **Claude Sonnet** because numeric accuracy matters more than speed there: a wrong dollar figure in an answer permanently damages trust, while a half-second of extra latency on a deliberate question does not.

## Building with Claude Code

Each screen is built as an isolated session. Claude Code only needs to read:

1. `CLAUDE.md` (the map)
2. `styles/tokens.css` (design tokens)
3. `public/prompts/penny-system.md` (voice core)
4. `screen-briefs/0X-{screen}.md` (scoped spec)

That keeps each build session's context under ~1000 lines instead of loading the full product spec. See `CLAUDE.md` for the build order and the exact instructions for each screen.

## Running locally

```bash
cd demo
npm install
npm run dev
# open the URL Vite prints
```

The demo calls the live Cloudflare Worker at `penny-api.nikjain1588.workers.dev` with a demo token. No local Claude API key required.

Run the tests:

```bash
npm test
```
