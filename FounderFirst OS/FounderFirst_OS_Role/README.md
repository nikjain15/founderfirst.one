## How to Use

**Project setup (once per project):**
```
1. Copy the template from IDENTITY.md
2. Save as IDENTITY_[project].md (e.g. IDENTITY_penny.md)
3. Fill in every field
4. Update after every locked decision or stage change
```

**Standard session:**
```
1. Paste your project's IDENTITY file (e.g. IDENTITY_penny.md)
2. Paste the role file you need
3. Write: ASK: [your specific situation in 1–2 sentences]
```

**High-stakes decision:**
```
Same as above, but end your ask with: + CRITIC
The role will give its primary response, then challenge it.
```

**Switching roles mid-session:**
Just say: `SWITCH TO: [role name]` — paste the new role file if it is a new chat.

## What Every Role Does

Every role file contains:
- **WHO I AM** — deep experience, philosophy, principles
- **MY SKILLS** — specific frameworks and methods, not generic advice
- **HOW I BEHAVE** — asks before building, gives strong opinions with trade-offs, stays concise, defers final decisions to the founder
- **HANDOFFS** — what it receives from and delivers to other roles
- **CRITIC MODE** — activated with `+ CRITIC` — challenges its own output

Every response ends with:
- `LEARN:` — concept used + mistake founders make here
- `NEXT:` — the single best question to ask next

---

## How Every Role Works

All roles follow these rules (from IDENTITY.md standing rules):
1. **Ask first.** Clarify scope, format, and expectations before producing work.
2. **Best thinking.** Recommend the best solution — scalable, trusted, loved. Do not constrain by team size or resources.
3. **Trade-offs, not gatekeeping.** Present options with trade-offs. Never say "do not do X" — say "here is X, here is the trade-off, here is my recommendation."
4. **Founder decides.** Every role advises. The founder is the CEO and makes the call.
5. **Concise.** Do exactly what is asked. No extra work that was not requested.

## Multi-Project Setup

Each project gets its own IDENTITY file. The roles are general-purpose experts — the IDENTITY file makes their advice specific.

- `IDENTITY_penny.md` → roles give Penny-specific advice
- `IDENTITY_[next project].md` → roles give that project's advice
- Same roles, different context, different recommendations

## One Rule

Update your project's IDENTITY file every time you lock a decision or your stage changes.
