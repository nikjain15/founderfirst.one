## Planned research (not yet executed)

From `BookKeeping/product/spec-brainstorm-decisions.md` research pre-commits:

1. **Prevalence survey (200 solo freelancers)** — CPA vs. DIY, tools tried and abandoned, entity-type distribution, S-Corp penetration by revenue band.
2. **Diary study (8–10 Alexes, 14 days)** — primary landing preference, emotional patterns around bookkeeping, re-entry behaviour after gaps.
3. **S-Corp freelancer interviews (6–8 users)** — what S-Corp-elected solo freelancers need; what breaks in QBO for them today.
4. **Concept test on approval-card fatigue (4-week prototype)** — retention curve weeks 2–4.
5. **Competitive churn interviews (8–10 users)** — users who left Keeper / FlyFin / QBSE / Wave.
6. **CPA interviews (5 CPAs)** — do they want a Penny CPA view?

When any of these is commissioned, its protocol and findings should be saved in this folder with a row added to the table above.

## How to add new research

When starting a new research workstream for the solo-freelancer segment:
1. If it fits an existing theme, add to the relevant file.
2. If it's a new topic, create a new `.md` file at this level with a clear name.
3. Add a row in the appropriate table above.
4. If the research blocks product decisions, reference it from `BookKeeping/product/spec-brainstorm-decisions.md`.
